# ai-config-authoring

Vendor-neutral craft knowledge for writing effective AI-agent
configuration — skills, rules, agents, and always-on instructions — for
any AI coding client (ten are covered by name, as of 2026 — Claude Code,
OpenCode, GitHub Copilot, Codex, Cursor, Kiro, JetBrains Junie, Gemini
CLI, Zed, and Amp).

The skill guides the decisions that make config actually fire and stay
useful: choosing between a skill, rule, hook, or always-on instruction;
writing descriptions that trigger; budgeting context; structuring
progressive disclosure; and designing guardrails that survive contact
with real sessions.

## Install

```sh
grim add ghcr.io/grimoire-rs/skills/ai-config-authoring:0
```

## What's inside

- `SKILL.md` — decision index and authoring workflow
- `references/choosing-types.md` — skill vs rule vs agent vs instruction
- `references/descriptions.md` — descriptions that trigger reliably
- `references/skill-design.md`, `rule-design.md`, `agent-design.md`
- `references/guardrails.md`, `checklist.md`
- `references/updating.md` — the maintainer re-research protocol
